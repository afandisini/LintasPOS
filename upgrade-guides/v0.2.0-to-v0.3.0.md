# Upgrade Guide: v0.2.0 -> v0.3.0

## Scope
- Track type: `MINOR` (backward-compatible by contract).
- Core-only patch policy: changes must target `system/`, `bootstrap/`, `public/`, or root tooling.
- User-owned paths (`app/`, `routes/`, `database/`) must not be overwritten by framework updater.

## Required Pre-Check
1. `php aiti upgrade:check --from=v0.2.0 --target=v0.3.0`
2. Review:
   - breaking list
   - risk list
   - conflict scan

## Apply Procedure
1. Dry-run first:
   - `php aiti upgrade:apply --from=v0.2.0 --target=v0.3.0`
2. Real apply:
   - `php aiti upgrade:apply --from=v0.2.0 --target=v0.3.0 --apply`
3. Verify:
   - `composer test`
   - `php aiti list`
   - `php aiti migrate status`

## Backup Policy
- Touched files are backed up to `*.bak.YmdHis` before overwrite/merge.
- If rollback is needed, restore from the newest backup.

## Migration Policy
- Use incremental migrations only.
- Destructive change must include explicit backup instructions.
- Ensure `migrate status` and `migrate rollback --step=1` remain healthy.

## Notes
- This guide is scaffolded. Add concrete patch entries to `upgrade-guides/index.php` when an actual release delta is finalized.
